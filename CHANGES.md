# CHANGES.md — إضافة Riddles API (ألغاز)

هذا الملف يوثّق التغييرات التي تمت في هذه الدفعة فقط. انسخي كل ملف إلى
نفس المسار النسبي بالضبط في مشروعك الحالي (استبدال أو إضافة حسب النوع
المذكور). **لم يتم لمس أي كود خاص بـ Quiz أو True/False إطلاقًا.**

---

## ملخص التغيير

تمت إضافة لعبة جديدة كاملة **Riddles (ألغاز)** للمشروع، بنفس أسلوب
وتنظيم الألعاب الموجودة فعليًا (خصوصًا `true-false`)، بدون إنشاء مشروع
جديد وبدون تعديل أي كود خاص بـ `quiz` أو `true-false`.

- **300 لغز** حقيقي بالعربي (من ملف `data.json` الذي رفعتِه)، كل لغز
  بحقل `answers` كمصفوفة إجابات مقبولة (نفس أسلوب الكويز)، مع `id`
  فريد **مرقّم تسلسليًا (1 إلى 300) ومخزَّن مسبقًا داخل ملف JSON نفسه**
  حسب طلبك، وليس مولَّدًا في وقت التشغيل.
- لعبة الألغاز متاحة على **نفس المسار العام المشترك بين كل الألعاب**:
  `/api/games/riddles/...` — بنفس نمط `quiz` و`true-false` تمامًا،
  بدون أي مسار مختصر إضافي وبدون أي تعديل على `next.config.ts` (تم
  التأكد أن `count` و`:id` و`random-exclude` كلها موجودة وتعمل فعليًا
  ضمن البنية العامة `/api/games/[game]/...` الحالية دون أي حاجة
  لإضافات).

---

## قرار مهم اتخذته أثناء التنفيذ (يحتاج مراجعتك)

ملف `data.json` الذي رفعتِه يحتوي حقل **`answers` كمصفوفة إجابات**
لكل لغز (مثل `["حفرة", "الحفرة"]`)، وليس حقل `answer` كنص واحد كما ورد
في نص الطلب. بعد سؤالك، اخترتِ الإبقاء على `answers` كمصفوفة (نفس
أسلوب الكويز)، للحفاظ على كل الصياغات الصحيحة المقبولة للإجابة. لو
غيّرتِ رأيك لاحقًا وعايزة `answer` كنص واحد، هذا يتطلب تعديل 3 ملفات
فقط: `riddles.types.ts`، `riddles.validation.ts`، وإعادة توليد
`questions.json`.

## الـ Endpoints الجديدة

كل ما يلي متاح على `/api/games/riddles/...`:

| Endpoint | الوصف |
|---|---|
| `GET /api/games/riddles/random` | لغز عشوائي واحد |
| `GET /api/games/riddles/random-exclude?ids=1,5,8,20` | لغز عشوائي مع استثناء ids (anti-repeat) |
| `GET /api/games/riddles/all` | كل الألغاز (300) |
| `GET /api/games/riddles/count` | العدد الإجمالي |
| `GET /api/games/riddles/:id` | لغز واحد بواسطة id |

مثال استجابة (`GET /api/games/riddles/random`):

```json
{
  "success": true,
  "data": {
    "id": 1,
    "question": "شيء كلما أخذت منه كبر، ما هو؟",
    "answers": ["حفرة", "الحفرة"]
  }
}
```

مثال خطأ (id غير موجود):

```json
{
  "success": false,
  "message": "No item with id 999999 in \"riddles\"",
  "code": "NOT_FOUND"
}
```

## تقرير استيراد البيانات

- **المصدر**: `data.json` (رفعتِه)
- **عدد السجلات الأصلية**: 300
- **السجلات غير الصالحة**: 0 (كل سجل فيه `question` نص غير فارغ و`answers` مصفوفة غير فارغة من نصوص غير فارغة)
- **السجلات المتجاهَلة**: 0
- **الـ id**: غير موجود في المصدر الخام أصلًا — تم توليده وإضافته **داخل ملف JSON نفسه** بشكل تسلسلي (1 إلى 300) حسب ترتيب كل لغز في الملف الأصلي، حسب طلبك الصريح، وليس في وقت التشغيل.
- **العدد النهائي المُسلَّم**: 300 لغز، بدون أي بيانات تجريبية/placeholder.

## نظام منع التكرار (Anti-Repeat)

لعبة الألغاز تتبع بالضبط نفس النمط المستخدم في `quiz` و`true-false`:

1. البوت يحتفظ بقائمة أرقام (`id`) الألغاز المستخدمة لكل مجموعة.
2. البوت يستدعي `GET /api/games/riddles/random-exclude?ids=<الأرقام المستخدمة>` بدل `/random` العادي.
3. البوت يضيف `id` اللغز المُرجَع لقائمته.
4. لو رجع الـ endpoint خطأ `404 NOT_FOUND` (كل الألغاز استُنفدت)، يصفّر البوت القائمة ويحاول تاني.

كود جاهز كامل لهذا النمط موجود في [`docs/riddles.md`](./docs/riddles.md#منع-تكرار-نفس-اللغز-anti-repeat).
