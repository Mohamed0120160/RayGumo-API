# CHANGES.md — إضافة Emoji API (خمّن الشخصية من الإيموجي)

هذا الملف يوثّق التغييرات التي تمت في هذه الدفعة فقط. انسخي كل ملف إلى
نفس المسار النسبي بالضبط في مشروعك الحالي (استبدال أو إضافة حسب النوع
المذكور). **لم يتم لمس أي كود خاص بـ Quiz أو True/False أو Riddles أو
Eye إطلاقًا.**

---

## ملخص التغيير

تمت إضافة لعبة جديدة كاملة **Emoji Game (خمّن الشخصية من الإيموجي)**
للمشروع، بنفس أسلوب وتنظيم الألعاب الموجودة فعليًا (خصوصًا `riddles`
و`eye`)، بدون إنشاء مشروع جديد وبدون تعديل أي كود خاص بالألعاب الأخرى.

- **500 سؤال** حقيقي (من ملف `data.json` الذي رفعتِه، بعد إصلاحه — انظر
  [تقرير استيراد البيانات](#تقرير-استيراد-البيانات) بالأسفل)، كل سؤال
  بحقل `emoji` (الإيموجيات)، `category` (بدل `anime` حسب طلبك)، و
  `answers` كمصفوفة إجابات مقبولة (عربي/إنجليزي معًا)، مع `id` فريد
  **مرقّم تسلسليًا (1 إلى 500) ومخزَّن مسبقًا داخل ملف JSON نفسه**، وليس
  مولَّدًا وقت التشغيل.
- لعبة "إيموجي" متاحة على **نفس المسار العام المشترك بين كل الألعاب**:
  `/api/games/emoji/...` — بنفس نمط `quiz` و`riddles` و`eye` تمامًا،
  بدون أي مسار مختصر إضافي وبدون أي تعديل على `next.config.ts` (تم
  التأكد أن `count` و`:id` و`random-exclude` كلها موجودة وتعمل فعليًا
  ضمن البنية العامة `/api/games/[game]/...` الحالية دون أي حاجة
  لإضافات).

---

## قرار مهم اتخذته أثناء التنفيذ (يحتاج مراجعتك)

طلبك الأصلي حدّد شكل endpoints مختلفًا تمامًا عن بنية المشروع الفعلية:
مسارات مستقلة بالشكل `/api/emoji/random`، `/api/emoji/random-exclude?exclude=1,5,8`،
`/api/emoji/all`، `/api/emoji/count`، و`/api/emoji/[id]`.

لكن مشروع RayGumo API الحالي **لا يستخدم مسارات مستقلة لكل لعبة**. بدل
ذلك، البنية الفعلية تعتمد **مسارًا عامًا واحدًا مشتركًا بين كل الألعاب**:
`/api/games/[game]/...`، مع "سجلّ ألعاب" مركزي (`registry.ts`) يوجّه كل
طلب حسب اسم اللعبة في الرابط. هذا هو نفس الأسلوب المستخدم بالضبط في
`quiz` و`true-false` و`riddles` و`eye` — لا يوجد أي منها على مسار مستقل
من نوع `/api/quiz/...` مثلًا.

بناءً على تعليماتك الصريحة بالالتزام الكامل بنفس معايير وبنية المشروع
الحالي وعدم تغيير أي API موجودة وعدم إنشاء تنسيقات جديدة، **اخترت اتّباع
البنية العامة الفعلية بدل المسارات المستقلة المذكورة حرفيًا في طلبك**.
النتيجة العملية:

| في طلبك | البنية الفعلية المطبّقة | السبب |
|---|---|---|
| `GET /api/emoji/random` | `GET /api/games/emoji/random` | نفس نمط كل الألعاب الأخرى (`/api/games/[game]/...`) |
| `GET /api/emoji/random-exclude?exclude=1,5,8` | `GET /api/games/emoji/random-exclude?ids=1,5,8` | اسم المعامل الموحّد في كل المشروع هو `ids`، وليس `exclude` |
| `GET /api/emoji/all` | `GET /api/games/emoji/all` | نفس السبب |
| `GET /api/emoji/count` | `GET /api/games/emoji/count` | نفس السبب |
| `GET /api/emoji/[id]` | `GET /api/games/emoji/:id` | نفس السبب |

شكل الاستجابة (`{ success, data }` و`{ success: false, message, code }`)
وكل رسائل/أكواد الأخطاء (`NOT_FOUND`, `BAD_REQUEST`, `INTERNAL_ERROR`)
**مطابقة تمامًا** لما طلبتِه ولما هو مستخدم فعليًا في المشروع — لم يتغيّر
شيء هنا.

لو كنتِ تقصدين فعلًا مسارات مستقلة منفصلة (`/api/emoji/...` بمعزل عن
`/api/games/...`)، هذا ممكن كإضافة route منفصلة لاحقًا (route جديد يستدعي
نفس `registry.ts`/`emoji.service.ts` الموجودين بدون تكرار أي منطق)، فقط
أخبريني.

## تقرير استيراد البيانات

- **الملف المصدر**: `data.json` (رفعتِه أنتِ).
- **مشكلة تحليل (parse) خطيرة موجودة في الملف الخام**: الملف لم يكن
  مصفوفة JSON واحدة صالحة، بل **6 مصفوفات JSON منفصلة ملتصقة ببعضها**
  (نمط `][` متكرر عند الأسطر 302، 603، 1204، 1805، 2406 من الملف
  الأصلي) — على الأرجح نتيجة دمج عدة دفعات تصدير بيانات دون فاصلة بينها.
  هذا لم يكن خطأ تحقق (validation)، بل خطأ syntax في الملف نفسه يمنع أي
  محلل JSON من قراءته كملف واحد.
- **الإصلاح**: تم تقسيم الملف عند كل نقطة `][`، تحليل كل قطعة كمصفوفة
  مستقلة صالحة، ثم دمج كل القطع الست في مصفوفة واحدة نهائية بنفس ترتيب
  الظهور الأصلي.
- **إعادة تسمية الحقل**: تم استبدال `"anime"` بـ `"category"` في كل
  سجل من الـ 500، حسب طلبك الصريح.
- **إجمالي السجلات بعد الدمج**: 500.
- **سجلات غير صالحة موجودة**: 0 (كل سجل يحتوي `id` رقميًا موجبًا،
  `emoji` وكلاهما `category` نصوصًا غير فارغة، و`answers` مصفوفة غير
  فارغة من نصوص غير فارغة).
- **سجلات تم تجاهلها**: 0.
- **تكرار في الـ id**: لا يوجد — كل الـ 500 قيمة `id` فريدة، متسلسلة من
  1 إلى 500 بالضبط، وبنفس ترتيب ظهورها في الملف الأصلي (لم يُعَد ترقيمها
  أو ترتيبها).
- **العدد النهائي المشحون**: 500 سؤال.
