# لعبة رتب (Sort) — توثيق كامل

هذا الملف يوثّق لعبة "رتب" بالتفصيل: بنية البيانات، كل الـ endpoints المتاحة، أمثلة الطلبات والاستجابات، نظام الـ `id`، آلية منع التكرار (`random-exclude`)، وحالات الأخطاء.

هذه اللعبة مبنية بنفس بنية لعبة [عين](./eye.md) و[الألغاز](./riddles.md) بالضبط (نفس الـ endpoints، نفس شكل الاستجابة، ونظام `id` مخزَّن مسبقًا في الملف). الفرق الوحيد الجوهري: استجابة `random` و`random-exclude` هنا **لا تحتوي أبدًا** على الإجابة الصحيحة — راجعي [لماذا answers غير موجودة في random](#لماذا-answers-غير-موجودة-في-random-وrandom-exclude) بالأسفل.

## وصف اللعبة

"رتب" لعبة تخمين نصية: يُعرض على اللاعب سطر واحد من الحروف المبعثرة لكلمة معيّنة (مثل `"ن ا ج ي ن د ا"`)، وعليه إعادة ترتيب هذه الحروف للوصول إلى الكلمة الصحيحة (`"ناجيندا"`).

كل سؤال في مصدر البيانات يحتوي **أكثر من ترتيب عشوائي مختلف** لنفس حروف الكلمة (عادة 3-4 أسطر)، وعند طلب سؤال عشوائي يختار الـ API **سطرًا واحدًا فقط** من بينها ويرسله - بحيث حتى لو ظهر نفس الـ `id` مرة أخرى مستقبلًا، قد يظهر بترتيب حروف مختلف.

الـ API هنا **مزوّد محتوى فقط** — لا يتحقق من إجابة المستخدم ولا يحتفظ بأي حالة (state) عن تقدّم اللعبة أو نقاط الخبرة أو الترتيب؛ هذا كله مسؤولية البوت المستهلك للـ API. الـ API بلا حالة (stateless) تمامًا.

## بنية البيانات

مصدر البيانات الأساسي هو ملف:

```
src/data/sort/questions.json
```

عبارة عن مصفوفة JSON تحتوي حاليًا **492 سؤالًا**.

شكل كل عنصر كما يُخزَّن في questions.json:

```json
{
  "id": 419,
  "questions": [
    "ن ا ج ي ن د ا",
    "د ن ا ج ي ن ا",
    "ي ا ن د ا ج ن",
    "ج ا د ن ا ي ن"
  ],
  "answers": ["ناجيندا"]
}
```

| الحقل | النوع | الوصف |
|---|---|---|
| `id` | `number` | معرّف فريد وثابت، **مخزَّن مسبقًا داخل questions.json نفسه** (وليس مولَّدًا وقت التحميل). انظر [نظام الـ id](#نظام-الـ-id) بالأسفل. |
| `questions` | `string[]` | مصفوفة تحتوي أكثر من ترتيب عشوائي مختلف لحروف نفس الكلمة. عند اختيار سؤال عشوائي (`random` أو `random-exclude`) يُختار **عنصر واحد فقط** منها عشوائيًا ويُرسَل كحقل `question` مفرد. |
| `answers` | `string[]` | مصفوفة الإجابات المقبولة (الكلمة الصحيحة بدون تبعثر). قد تحتوي أكثر من إجابة صحيحة (تهجئات مختلفة لنفس الاسم)، بنفس نمط `answers` في [Quiz API](./quiz.md). **لا تظهر أبدًا** في استجابات `random`/`random-exclude` — راجعي القسم التالي. |

أي سجل ناقص فيه `id` (رقم صحيح موجب فريد) أو `questions` (مصفوفة نصوص غير فارغة) أو `answers` (مصفوفة نصوص غير فارغة) يُتجاهل تلقائيًا وقت التحميل ولن يظهر في أي endpoint - دون أن يوقف الـ API بالكامل. لو اتكرر نفس الـ `id` بين عنصرين، أول عنصر بيفوز والباقي بيتجاهل مع تحذير في الـ logs.

---

## نظام الـ id

- الـ `id` **رقم صحيح موجب فريد**، **مخزَّن مسبقًا وصراحة داخل كل عنصر في `questions.json` نفسه** - بنفس أسلوب لعبتَي "عين" و"الألغاز" (على عكس الكويز حيث يُولَّد تلقائيًا حسب الترتيب وقت التحميل).
- عند إضافة أسئلة جديدة يدويًا للملف، **لازم تدي كل عنصر جديد `id` فريدًا لم يُستخدم من قبل** (عادة: أكبر `id` موجود حاليًا + 1). الـ API يتحقق فقط أن كل `id` رقم صحيح موجب وفريد؛ لا يولّد أي `id` بنفسه.
- لو اتكرر نفس الـ `id` بين عنصرين في الملف بالخطأ، أول عنصر بترتيب الظهور يفوز، والباقي يُتجاهل مع تحذير في الـ logs (`console.warn`) يساعد وقت مراجعة عملية استيراد أسئلة جديدة.
- نفس الـ `id` يُستخدم في نظام منع التكرار (`random-exclude`) بالضبط بنفس أسلوب باقي الألعاب.

## لماذا answers غير موجودة في random وrandom-exclude

على عكس كل الألعاب الأخرى في المشروع (Quiz، Eye، Riddles...) التي ترجع العنصر كاملًا بما فيه `answers` حتى في `random`، لعبة "رتب" **تحذف** حقل `answers` عمدًا من استجابات `GET /api/games/sort/random` و`GET /api/games/sort/random-exclude`.

**السبب:** لو رجعت `answers` مع السؤال، ستكون الإجابة الصحيحة موجودة فعليًا داخل نفس رسالة الـ API التي يستقبلها بوت الواتساب لعرض السؤال على اللاعبين - ما يعني تسريب الإجابة بسهولة (سواء عبر خطأ برمجي في البوت يعرضها بالغلط، أو عبر أي طرف يراقب استجابات الشبكة). لذلك شكل الاستجابة في هذين الـ endpoint تحديدًا هو **فقط**:

```json
{ "id": 419, "question": "ن ا ج ي ن د ا" }
```

أما `GET /api/games/sort/all` و`GET /api/games/sort/:id` فيرجعان العنصر **كاملًا** بما فيه `answers` ومصفوفة `questions` كاملة - لأن الغرض منهما أدوات إدارة المحتوى والمراجعة، وليس تشغيل اللعبة الفعلي أمام اللاعبين.

> **مهم لمطوّري البوت:** بما أن `random`/`random-exclude` لا يرجعان الإجابة الصحيحة، على البوت جلب الإجابة الصحيحة بشكل منفصل عبر `GET /api/games/sort/:id` (باستخدام نفس الـ `id` الذي استلمه) إذا احتاج التحقق منها، أو الاحتفاظ بنسخة محلية من `answers` عند استخدام `all` مسبقًا.

## الـ Endpoints

جميع الـ endpoints التالية متاحة على المسار العام المشترك بين كل الألعاب: `/api/games/sort/...` (نفس نمط `quiz` و`eye` و`riddles` تمامًا).

### `GET /api/games/sort/random`

ترجع سؤال "رتب" عشوائيًا واحدًا: `id` + سطر حروف واحد مُختار عشوائيًا من مصفوفة `questions` الخاصة به. **بدون** `answers`.

**مثال الطلب:**

```bash
curl https://raygumo-api.vercel.app/api/games/sort/random
```

**مثال الاستجابة (200):**

```json
{
  "success": true,
  "data": {
    "id": 419,
    "question": "ن ا ج ي ن د ا"
  }
}
```

---

### `GET /api/games/sort/random-exclude`

ترجع سؤالًا عشوائيًا واحدًا (بنفس شكل `random` المختصر)، مع استثناء أي `id` موجود ضمن قائمة معامل الاستعلام `ids` (مفصولة بفواصل). هذا هو آلية منع التكرار (anti-repeat) في اللعبة.

`GET /api/games/sort/random-exclude?ids=1,5,8,20`

**لماذا هذا الـ endpoint موجود:** الـ API بلا حالة تمامًا - لا يخزّن أي معلومة دائمة عن الأسئلة التي عُرضت سابقًا. البوت (مثلًا بوت واتساب) هو المسؤول عن تتبّع الأسئلة المستخدمة لكل مستخدم/مجموعة، ثم إرسال قائمة تلك الـ `id` مع كل طلب حتى لا يتكرر نفس السؤال.

**مثال الطلب:**

```bash
curl "https://raygumo-api.vercel.app/api/games/sort/random-exclude?ids=1,5,8,20"
```

**مثال الاستجابة (200) — سؤال غير موجود ضمن `1,5,8,20`:**

```json
{
  "success": true,
  "data": {
    "id": 42,
    "question": "و ك غ و"
  }
}
```

**سلوكيات خاصة:**

- إذا كان معامل `ids` غير موجود أو فارغًا، يعمل هذا الـ endpoint تمامًا مثل `/random` العادي (استثناء لا شيء = كل الأسئلة متاحة).
- إذا استُثنيت **كل** الأسئلة المتاحة (لم يتبقَّ أي سؤال جديد)، يرجع الـ endpoint خطأ `404 NOT_FOUND` واضحًا بدل تكرار سؤال قديم بصمت. على البوت عندها تصفير قائمة الأسئلة المستخدمة والمحاولة من جديد.

---

### `GET /api/games/sort/all`

ترجع كل أسئلة "رتب" كمصفوفة كاملة (492 سؤالًا حاليًا)، **بما فيها** `questions` كاملة و`answers`.

**مثال الطلب:**

```bash
curl https://raygumo-api.vercel.app/api/games/sort/all
```

**مثال الاستجابة (200) — مختصرة:**

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "questions": ["ف ي و ل", "و ل ي ف", "ي ف ل و", "ل ف و ي"],
      "answers": ["لوفي"]
    },
    {
      "id": 2,
      "questions": ["ت و ر ا ن", "ر ن ا ت و", "ا و ت ر ن", "ن ت ر ا و"],
      "answers": ["ناروتو"]
    }
  ]
}
```

---

### `GET /api/games/sort/count`

ترجع العدد الإجمالي لأسئلة "رتب" المتاحة حاليًا.

**مثال الطلب:**

```bash
curl https://raygumo-api.vercel.app/api/games/sort/count
```

**مثال الاستجابة (200):**

```json
{
  "success": true,
  "data": {
    "count": 492
  }
}
```

---

### `GET /api/games/sort/:id`

ترجع سؤالًا واحدًا محددًا بالضبط عبر رقمه (`id`)، **كاملًا** بما فيه `questions` و`answers` - بدل عنصر عشوائي مختصر.

**مثال الطلب:**

```bash
curl https://raygumo-api.vercel.app/api/games/sort/419
```

**مثال الاستجابة (200):**

```json
{
  "success": true,
  "data": {
    "id": 419,
    "questions": [
      "ن ا ج ي ن د ا",
      "د ن ا ج ي ن ا",
      "ي ا ن د ا ج ن",
      "ج ا د ن ا ي ن"
    ],
    "answers": ["ناجيندا"]
  }
}
```

**مثال استجابة عند عدم وجود العنصر (404):**

```json
{
  "success": false,
  "message": "No item with id 999999 in \"sort\"",
  "code": "NOT_FOUND"
}
```

**مثال استجابة عند `id` غير صالح (400):**

```json
{
  "success": false,
  "message": "Invalid id parameter: \"abc\"",
  "code": "BAD_REQUEST"
}
```

## شكل الاستجابة الموحّد

كل endpoint في هذه اللعبة يتّبع نفس الشكل الموحّد المستخدم في كل أنحاء المشروع (راجعي `src/types/api.ts`):

**نجاح:**

```json
{
  "success": true,
  "data": { }
}
```

**خطأ:**

```json
{
  "success": false,
  "message": "نص يشرح الخطأ",
  "code": "NOT_FOUND"
}
```

يعني كود البوت ممكن يعمل تحقّق واحد بس (`json.success`) يغطي كل الحالات.

---

## حالات الأخطاء

| الحالة | HTTP Status | code |
|---|---|---|
| `id` غير موجود ضمن أسئلة "رتب" | 404 | `NOT_FOUND` |
| كل الأسئلة مستثناة في `random-exclude` (لم يتبقَّ سؤال جديد) | 404 | `NOT_FOUND` |
| `id` في الرابط غير رقم صحيح موجب (مثل `abc` أو `-1`) | 400 | `BAD_REQUEST` |
| قيمة داخل معامل `ids` غير رقم صحيح صالح | 400 | `BAD_REQUEST` |
| خطأ غير متوقع في الخادم | 500 | `INTERNAL_ERROR` |

## إضافة/تحديث أسئلة رتب

لتحديث المحتوى: عدّلي `src/data/sort/questions.json` مباشرة، تأكدي أن كل سؤال جديد له `id` فريد لم يُستخدم من قبل و`questions` تحتوي أكثر من ترتيب حروف صالح لنفس `answers`، ثم اعملي commit وارفعي (redeploy) - بدون أي تعديل كود.
